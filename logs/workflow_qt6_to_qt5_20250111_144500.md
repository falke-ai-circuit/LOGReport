## Learnings: Consolidated from all phases

### Patterns
1. **Enum-to-Attribute Conversion**: PyQt6 uses strongly-typed enums (QPalette.ColorRole.X, Qt.GlobalColor.Y) while PyQt5 uses direct attributes (QPalette.X, Qt.Y). Regex pattern: `QPalette\.ColorRole\.` → `QPalette.`, `Qt\.GlobalColor\.` → `Qt.`

2. **5-Step Migration Pattern**:
   - Step 1: Imports (PyQt6 → PyQt5)
   - Step 2: Enums (ColorRole.X → X, GlobalColor.Y → Y)
   - Step 3: Methods (exec() → exec_())
   - Step 4: Import paths (QtGui.QAction → QtWidgets.QAction)
   - Step 5: Runtime paths (PyQt6/Qt6 → PyQt5/Qt5)

3. **Staged Replacement Strategy**: Separating imports from API changes isolates syntax errors from behavioral issues, enabling faster debugging

4. **Bulk Regex with Targeted Fixes**: PowerShell bulk replacements handle 95% of changes, manual fixes for critical 5% (exec(), QAction) ensure correctness

### Approaches
1. **Multi-pattern regex replacement**: Systematic approach reduces manual effort while maintaining accuracy
2. **Import-first methodology**: Fixing imports before API changes isolates syntax from behavioral errors
3. **Test-driven validation**: Running tests incrementally confirms each migration step
4. **Direct code execution**: Using mcp_pylance to validate imports without creating temporary files
5. **Memory-first workflow**: Loading global + project memory before starting provides context for similar patterns

### Methodologies
1. **CEPH Evolution Tracking**: Context progresses from Initial (facts) → Mid-Phase (analysis) → Final (validation)
2. **Parallel Testing**: Having both PyQt5 and PyQt6 installed enables incremental validation
3. **Codegraph-Driven Analysis**: Loading entire codegraph.json in ASSESS phase enables dependency tracing throughout workflow

## Patterns: Reusable approaches + methodologies

### Migration Pattern (Reusable for PyQt version changes)
```
1. IMPORTS: Find/replace module names (PyQt6 → PyQt5)
2. ENUMS: Convert enum-style to attribute-style (ColorRole.X → X)
3. METHODS: Rename deprecated methods (exec() → exec_())
4. IMPORTS_PATHS: Update moved classes (QtGui.QAction → QtWidgets.QAction)
5. RUNTIME: Update plugin/resource paths in build scripts
```

### Validation Strategy
```
1. Syntax check with Pylance (mcp_pylance_mcp_s_pylanceFileSyntaxErrors)
2. Import verification (mcp_pylance_mcp_s_pylanceRunCodeSnippet)
3. Base class inspection (verify inheritance chain)
4. Unit test execution (pytest on PyQt-specific tests)
5. Manual integration test (run GUI, verify visual appearance)
```

### Risk Mitigation
```
1. Keep both Qt versions installed during migration
2. Use bulk replacements for systematic changes (imports, enums)
3. Manual review for critical changes (exec(), import paths)
4. Incremental testing after each migration step
5. Pre-existing PyQt5.spec file reduces build configuration risk
```
